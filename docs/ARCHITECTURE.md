# Architecture — ai-redact

Local-only PDF redaction pipeline. For day-to-day operation see
[RUNBOOK.md](RUNBOOK.md).

## Layout

```
ai-redact/
├── src/
│   ├── redact.py           # the entire tool (detection → redaction → verify → report)
│   └── make_sample_pdf.py  # fake-data test PDF generator (writes to input/)
├── scripts/
│   ├── run.sh              # setup + batch driver: input/ → output/
│   └── redact.sh           # single-file wrapper: runs src/redact.py with .venv's python
├── config/
│   ├── redact_config.example.yaml  # committed template
│   └── redact_config.yaml          # user's personal copy (gitignored; auto-created
│                                   # by run.sh; loaded by DEFAULT on every run,
│                                   # overridable with --config)
├── input/                  # drop zone for documents (contents fully gitignored;
│                           # fake test samples generated by make_sample_pdf.py)
├── output/                 # results; EMPTIED at the start of every batch run
├── docs/                   # this documentation
└── .venv/                  # created by scripts/run.sh on first run
```

## Design principles

1. **Local only.** No network calls anywhere. Dependencies: PyMuPDF, PyYAML,
   zxing-cpp/Pillow (QR/barcodes), and Tesseract language data (OCR) — all
   installed automatically by `scripts/run.sh`.
2. **True redaction.** PyMuPDF redaction annotations + `apply_redactions()`
   delete the text and blank overlapping image pixels. Never a box drawn
   over intact content.
3. **Precision over recall for finances.** Numeric patterns require context
   (a nearby label) so balances, prices, tickers, cost basis, and ordinary
   dates are never touched. Only DOB-labeled dates are redacted.
4. **Fail loudly, never fake.** Unreadable scanned pages produce warnings /
   exit code 2 — never a silently "clean" file.
5. **Names are explicit.** No NLP/NER guessing; names come only from
   `custom_terms` in the config. Deterministic and auditable.

## Pipeline (src/redact.py)

```
open PDF
└─ document level:
   • remove embedded file attachments (invisible carriers of anything)
   • scrub bookmark/ToC titles that match any pattern → "[redacted]"
└─ per page:
   1. extract text (page.get_text)
   2. no text + has images?  → OCR (get_textpage_ocr, Tesseract via PyMuPDF)
      • OCR fails/unavailable → record as scanned_page, skip (warned later)
   3. delete hyperlinks whose URI matches any pattern (mailto:, tel:, …)
   4. run enabled CATEGORY_PATTERNS + custom terms over the text,
      minus exclude_terms (the config's never-redact allowlist)
   5. locate each match on the page (search_for; OCR textpage for scans)
      • unlocatable match → recorded, reported "<< NOT LOCATED", never silent
   6. decode QR/barcodes in images (zxing-cpp) → add their rects
   7. add redaction annotations for every rect
└─ apply_redactions(images=PIXELS)   # deletes text, blanks image regions
└─ scrub document metadata
└─ save (garbage collect + deflate)
└─ VERIFY: reopen the OUTPUT file, re-run all patterns over text (re-OCR
   the scan pages), link URIs, and bookmark titles
   → report PASS (0 matches) or FAIL
└─ write report (real runs mask values; dry runs show them in full)
└─ exit code: 0 ok · 2 unreadable page(s) not redacted · 3 verify failed
```

### Detection model

`CATEGORY_PATTERNS` maps category → list of pattern dicts:

- `regex` — compiled pattern.
- `group` — optional named group `redact`: match on context
  (`"DOB: 03/15/1980"`) but redact only the value. This is how labeled
  fields work without redacting the labels or nearby prose.
- `validator` — optional post-check to kill false positives, e.g. the ABA
  checksum for bare 9-digit routing numbers, the Luhn checksum for card
  numbers, or minimum-digit-count rules that stop words like "Summary"
  being taken for IDs.

Two invariants keep the patterns safe:

- **Label separators never cross line breaks** (`SEP = [ \t]*[:#.-]?[ \t]*`,
  not `\s*`). Otherwise a label ending one line captures the first word of
  the next line — this was a real bug found in testing: after redaction,
  `MRN:` at a line end matched the following line's `Member` label and made
  verification fail.
- **Bare number matches must be unambiguous** (8–17 digit runs excluded
  from `$`/decimal/comma contexts), so financial figures can't match.

`PRESETS` (financial / medical / legal / general) are just named category
lists; `--categories` overrides them. Custom terms become whole-word,
case-insensitive literal patterns in a `custom` category appended to any
preset; `exclude_terms` filters matches out at detection time (both in the
redaction pass and in verification, so an excluded term can't fail verify).

### Config resolution

`config/redact_config.yaml` is loaded on every run with no flag needed; the
path resolves against the project root, so `./scripts/redact.sh` works from any
directory. `--config <file>` substitutes another config, and a missing
explicit config is a hard error — the tool never silently redacts less
than the user asked for. `run.sh` creates the personal config from
`redact_config.example.yaml` on first run; the personal copy is gitignored.

Beyond `custom_terms`/`exclude_terms`/`custom_patterns`, the config
carries `preset` (default document type), `categories` (per-detector
always-on/always-off switches applied on top of the preset — skipped when
`--categories` gives an explicit list), and `options` (`ocr`,
`redact_barcodes` toggles). Full reference: [CONFIGURATION.md](CONFIGURATION.md).
Precedence: `--categories` > `--preset` > config `preset` > `general`,
then config `categories` switches adjust, custom terms always add,
exclude terms always filter.

### Scanned pages (OCR)

PyMuPDF ships Tesseract built in and needs only the language data
(`brew install tesseract` provides it; `find_tessdata()` locates it, or
honors `$TESSDATA_PREFIX`). Pages with images but no text layer are OCR'd
at 300 dpi; matches are located through the OCR textpage, whose rectangles
map back to page coordinates, and `apply_redactions` blanks those image
pixels. Verification re-OCRs those pages in the output file to confirm the
image no longer shows the sensitive text. OCR'd pages are also flagged in
the report since OCR accuracy is imperfect.

### Reports

One `*_report.txt` per document: counts per category, per-match detail,
warnings (unreadable pages, unlocated matches, image pages), and the
verification result. Real-run reports mask matched values
(`0********56`) so the report can sit safely next to the redacted PDF;
dry-run reports show full values for accuracy checking.

## Batch driver (scripts/run.sh)

Idempotent setup then a loop:

1. Create `.venv` + install requirements if missing.
2. Install Tesseract via Homebrew if OCR data is absent (non-fatal).
3. Create `config/redact_config.yaml` from the template if missing.
4. `mkdir -p input output`; abort helpfully if `input/` has no PDFs.
5. Empty `output/` (keeps `.gitkeep`).
6. Run `src/redact.py` per PDF → `output/<name>_redacted.pdf`; classify
   per-file exit codes (0 ok, 2 unreadable pages, 3 verify failed, else
   failure).
7. Print a batch summary; exit non-zero if anything wasn't fully redacted.

## Git hygiene

`.gitignore` blocks all `*.pdf` and `*_report.txt`, `input/*` and
`output/*` wholesale (except `.gitkeep`), and `config/redact_config.yaml`
(the personal list of sensitive terms) — neither documents nor the user's
term list can be committed by accident. Test samples are generated locally
by `make_sample_pdf.py`, never committed.
